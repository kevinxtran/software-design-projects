#Week7
### 1. Overview and Purpose

- Focus on computational needs for data acquisition, processing, and actuator control in robotics and autonomous systems.
- Key characteristics of autonomous systems:
    - Real-time data acquisition and processing.
    - Timely actuator control to execute tasks based on processed data.

### 2. Parallels Between Humans and Autonomous Systems

- Human Sensory System: Receives input from various sensors (eyes, ears, nose, etc.), processed by the brain for decision-making.
- Autonomous Systems:
    - Require sensors for task-specific data (e.g., for a self-driving car: lane boundaries, other vehicles, and surroundings).
    - Process only a limited subset of tasks humans perform, making system design more feasible.

### 3. Data Acquisition in Autonomous Systems

- Example: _Self-Driving Cars_ – Utilizes multiple sensors, such as:
    - **Lidar**: Light detection and ranging.
    - **Cameras**: Visual data.
    - **Radar Sensors**: Detects object distances and speed.
    - **GPS Sensors**: Location tracking.
    - **Ultrasonic Sensors**: Obstacle detection.
- Types of data include images, videos, text, and numeric data.
- Data production rates: Self-driving cars generate around **4 terabytes/day**.

#### Challenges in Data Acquisition:

- **Real-time Processing**: Systems must process massive data amounts swiftly.
- **Efficient Storage**: Better data storage structures for quick access.
- **Algorithms**: Effective algorithms for data sorting and processing are essential.
- **Reusable Software Modules**:
    - Abstractions for sensor data to avoid developing new software with each hardware update.
    - Software interfaces need to adapt to new hardware with minimal modifications.
- **Data Transformation**: Data from various sensors must be converted to suitable formats (e.g., binary) for computational operations.

### 4. Data Processing and Interpretation

- _Task-Specific Requirements_:
    - Different autonomous systems have distinct data processing needs (e.g., self-driving cars focus on traffic signs and lanes, social robots may focus on human expressions).
- _Self-Driving Car Example_:
    - Detects nearby vehicles, road boundaries, traffic signals, and conditions (e.g., snow, rain).
- **Real-Time Processing**: Decision-making must occur in milliseconds to avoid accidents.
    - **Human Reaction Times**:
        - Visual: ~0.25 seconds
        - Audio: ~0.17 seconds
        - Touch: ~0.15 seconds
    - Autonomous systems aim to match or exceed human reaction times.

#### Challenges in Data Processing:

- **Algorithm Efficiency**: High-performance algorithms are required for quick, accurate decisions.
- **Artificial Intelligence (AI) and Machine Learning (ML)**:
    - AI/ML techniques enhance data processing capabilities.
- **Component Communication**:
    - Efficient software design enables communication between system components to execute actions (e.g., braking upon detecting an obstacle).

### 5. Actuator Control in Autonomous Systems

- _Role of Actuators_: Execute decisions made by the processing unit (e.g., self-driving car actuators for braking, steering).
- **Analogy to Muscles and Joints**:
    - Actuators act like muscles and joints in humans, responding to commands from the central processing unit to perform movements.

### Summary

- **Real-Time Data Handling**: Essential for gathering, processing, and acting on data without delays.
- **Task-Specific Design**: Autonomous systems are specialized for specific tasks, simplifying design but increasing precision requirements.
- **Challenges**: Include high data volume, need for efficient algorithms, real-time communication, and rapid actuator response.
