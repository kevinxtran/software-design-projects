#Week6
#### **Overview of MVC Architecture:**

- **Model:** Represents data and application logic. For this app, it handles a list of fruit records, including operations like adding and deleting fruits.
- **View:** The TableView displays the fruit list, and the DetailView shows detailed fruit information when a row is selected.
- **Controller:** Manages interactions between the model and view. There are two view controllers:
    - **ViewController:** Manages the main fruit list, handles adding/removing fruit, and triggers the segue to the DetailView.
    - **DetailViewController:** Displays detailed information for a selected fruit by loading a web page based on the fruit name.

#### **Model Implementation:**

- **Fruit Object:**
    - Contains `fruitName` and `fruitDescription` properties, both of type `String`.
    - Constructor initializes a fruit with these attributes.
- **FruitModel Class:**
    - Stores a list of `Fruit` objects in an array (`fruitList`).
    - Provides functions:
        - `numberOfFruits`: Returns the number of fruits.
        - `getFruit(at index: Int)`: Returns a fruit from the list based on its index.
        - `addFruitRecord`: Adds a new fruit to the list.
        - `removeFruitRecord`: Removes a fruit from the list by its index.

#### **ViewController:**

- **Purpose:** Manages the TableView.
- **Delegates & DataSource:**
    - Implements `UITableViewDataSource` methods, which dictate how the TableView is populated and how it interacts with the model.
- **Key Functions:**
    - **`numberOfRowsInSection`:** Returns the number of rows in the TableView based on the number of fruits in the `FruitModel`.
    - **`cellForRowAt`:** Configures each row by creating a `UITableViewCell` instance and populating it with fruit data (name and description) from the `FruitModel`.

#### **Handling Add/Remove Fruit Operations:**

- **Adding a Fruit:**
    - **Alert View:** Displays an alert with text fields for the fruit's name and description.
    - On clicking **OK**, the entered data creates a new `Fruit` object, which is added to the `FruitModel`.
    - The TableView is updated with the new fruit by calling `insertRows` to add a row at the correct position.
- **Removing a Fruit:**
    - **Swipe-to-Delete:** Enabled using `canEditRowAt` (returns `true` to allow editing) and `editingStyleForRowAt` (sets the row's style to `delete`).
    - When a delete action is triggered, the corresponding fruit is removed from the `FruitModel` using `removeFruitRecord`, and the TableView is updated with `deleteRows`.

#### **Segue to Detail View:**

- **Prepare for Segue:**
    - **`prepare(for segue: sender:)`:** This method is called before transitioning to the `DetailViewController`.
    - It retrieves the selected fruit's data (name) based on the row index and passes it to the `DetailViewController`.

#### **Key Takeaways for Study/Review:**

- **MVC Pattern in Action:** Understand the separation of concerns between the model (data), view (UI), and controller (logic and interactions).
- **TableView Operations:** Be clear on how to set up and configure a TableView with custom cells. Focus on `numberOfRowsInSection` and `cellForRowAt`.
- **Model Interaction:** Learn how the view interacts with the model (data) through the ViewController, and how data is updated in the model and reflected in the view.
- **CRUD Operations:** Pay attention to the add and remove functionalities, especially how data changes in the model and how the TableView is updated.
- **Segue Implementation:** Practice preparing data to pass between views during a segue, which is a key part of navigating between screens in iOS apps.
