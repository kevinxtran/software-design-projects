#Week6
**Overview:**

- This case study focuses on a TableView application built using the Model-View-Controller (MVC) architecture for iOS.
- The application displays fruit information in a table view and provides basic functionality for adding, deleting, and viewing detailed information about fruits.

**Functionality:**

1. **Display Fruit Information in a TableView:**
    
    - The app shows fruit names and descriptions in a table.
    - A "Fruit Table" title appears at the top, with a "+" button to add new fruit records.
    - Placeholder for images, although not implemented in this case.
2. **Add Fruit Records:**
    
    - Clicking the "+" button opens a window to enter the fruit name and description.
    - The new fruit record is added to the table when the user confirms.
3. **Delete Fruit Records:**
    
    - Swiping left on a fruit row reveals a "Delete" button.
    - Clicking "Delete" removes the record from the table.
4. **Show Fruit Details:**
    
    - Clicking a row in the table opens a detail view where fruit information is fetched from Google search and displayed in a web view.

**UI Design:**

- The design uses **Xcode's Storyboard**, where UI elements are dragged and dropped.
- A **TableView** is used to display fruit records, with prototype cells for fruit name, description, and (potentially) images.
- **Accessory Views** (arrows) are added to rows for accessing the detail view.
- The **Segue** component connects the table view to the detail view, allowing data transfer between views.
- **Navigation Controller** is used to enable easy navigation between views, including automatic creation of a back button for returning to the main view.

**MVC Architecture Design:**

1. **Model:**
    
    - The **FruitModel** stores fruit data (name, description) and maintains a list of records.
    - Provides operations like add, remove, and fetch fruit records.
2. **View:**
    
    - The main view displays the TableView with the fruit records.
    - The detail view shows detailed information about a selected fruit.
    - Both views are controlled by their respective **ViewControllers**.
3. **Controller:**
    
    - **MainViewController** handles displaying the TableView, adding/removing fruit records, and managing interactions between the TableView and the Model.
    - **DetailViewController** displays detailed information about the selected fruit.
    - The controllers are responsible for the data flow, connecting the Model to the Views, and handling user actions (e.g., adding, deleting, selecting a fruit).

**Implementation Structure in Xcode:**

- **Main ViewController**: Manages the main table view and user interactions such as adding, deleting fruits.
- **Detail ViewController**: Displays the detailed view after selecting a fruit.
- **FruitModel**: Contains data and operations related to fruits.
- **FruitTableViewCell**: Defines the layout and behavior of each cell in the TableView.

**Class Diagram:**

- **FruitModel**: Holds fruit data and CRUD (Create, Read, Update, Delete) operations for fruit records.
- **MainViewController**: Controls the TableView, uses the FruitModel for data management, and triggers transitions to the DetailViewController.
- **DetailViewController**: Displays detailed information based on data passed from MainViewController.
- **FruitTableViewCell**: Represents each row in the TableView.

**Programming Language:**

- The iOS application is developed in **Swift**, with file extensions using `.swift` for each related file (ViewControllers, Model, etc.).

**Key Points to Remember:**

1. **MVC Structure**:
    
    - The **Model** manages data and logic (FruitModel).
    - The **View** is the UI (TableView, DetailView).
    - The **Controller** coordinates between the Model and View (MainViewController, DetailViewController).
2. **Navigation**: The **Navigation Controller** facilitates easy navigation between views, automatically adding a back button when transitioning between views.
    
3. **Data Flow**:
    
    - **MainViewController** interacts with the **FruitModel** for displaying and modifying data.
    - **DetailViewController** is fed data from the MainViewController using **segues**.
4. **User Actions**:
    
    - **Add** fruit through the "+" button, **delete** by swiping left, and **view details** by selecting a row.
5. **UI Configuration**:
    
    - Storyboards in Xcode are used to design views, with cells being dynamically generated based on fruit records.
    - Segue links are used to navigate between views, with data passed through the segue from the main view to the detail view.

**Summary:**

- TableView applications are common in mobile apps and follow the MVC architecture for organizing data, UI, and logic.
- The application discussed uses **FruitModel** for data management, **ViewControllers** for managing views, and **segues** for navigating between the main and detail views.
- Understanding how the controller manages interactions and data flow is essential for building similar apps.
