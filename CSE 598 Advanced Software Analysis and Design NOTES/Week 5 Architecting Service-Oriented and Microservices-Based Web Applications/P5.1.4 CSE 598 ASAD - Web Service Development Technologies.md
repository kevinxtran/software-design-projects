#Week5 
#### Overview of Web Service Development

- Web services require two key steps: **coding the service functionality** and **deploying the service on a server**.
- **Application developers** can access and use these services by retrieving the server's web service URL.

#### Major Technologies and Languages for Web Services

- **Java**: Commonly used with **Java Enterprise Edition** (Java EE) and **WebSphere** as a server platform.
- **.NET**: Uses **C#** for web services; typically deployed with **Microsoft IIS** (Internet Information Services).
- **Python**: Can be used with servers like **Apache** or its own web server options for web service deployment.
- **Angular**: Often mentioned for front-end integration with web services.

#### Class Focus on .NET and C# for Web Services

- **Course Focus**: .NET technologies, primarily C#.
- A recommended resource link compares **Java APIs** and **Microsoft .NET APIs** for similarities, differences, and strengths.

### Understanding .NET Technologies

#### .NET Frameworks Overview

- **.NET Core**: Cross-platform; supports Windows, Linux, and macOS applications.
- **.NET Framework**: Primarily for Windows-based applications.
- **Unified .NET (as of November 2020)**: Combines features of .NET Core and .NET Framework into a single, cross-platform framework.
    - Supports diverse application types: **web, mobile, desktop, game, and IoT**.

#### Development Environments (IDEs)

- **Visual Studio**: Main IDE for .NET development.
- **Other IDEs**:
    - **Visual Studio Code**: Lightweight, cross-platform IDE.
    - **Xamarin**: Specifically for iOS and mobile application development.

#### .NET Framework and Core Lifecycle

- While Microsoft released the **unified .NET** in 2020, **.NET Core** and **.NET Framework** will still be supported, though updates may be limited.

### Important .NET Technologies for Web Services

#### Types of Web Services

- This course covers both **REST** and **SOAP** web services using .NET technologies.

#### .NET Technologies in Focus

- **.NET Framework**: Selected for class use.
    - **Windows Communication Foundation (WCF)**: Framework used in the course for web service development.
    - **ASP.NET**: Also supports REST and SOAP services; offers more structured features like **MVC architecture**.
- **WCF vs. ASP.NET**:
    - **WCF**: Basic framework for web services, used in this course.
    - **ASP.NET**: More sophisticated, supports REST, SOAP, and MVC patterns.
