**Web API Calls from a Mobile Application: Key Concepts and Processing**

- **API Call Basics**:
    
    - Mobile applications make Web API calls to interact with web services.
    - Web services:
        - **REST** (Representational State Transfer) and **SOAP** (Simple Object Access Protocol) are common types.
        - REST uses the **HTTP** protocol, which is more widely adopted in mobile applications. SOAP runs over HTTP but is more complex.
- **REST API Components**:
    
    - **URI Resource**: The location where the web service is deployed.
    - **API Name**: Specifies which API function is being called.
    - **Parameters**: Passed to retrieve the appropriate data from the API.
- **Synchronous vs. Asynchronous Calls**:
    
    - **Synchronous**: The app waits for the API call to complete before proceeding (can block the app, causing poor user experience).
    - **Asynchronous**: The API call runs on a separate thread, allowing the main thread to remain active (recommended for better user experience).
    - Asynchronous calls are generally preferred for remote API calls since network delays are unpredictable.
- **Web API Data Retrieval Process**:
    
    - After the API call, the system processes the request (e.g., database search or computation) and returns the appropriate data via HTTP.
    - This data typically comes in **JSON** or **XML** format.
- **Data Formats: JSON vs. XML**:
    
    - **JSON**:
        - Lightweight and text-based.
        - Language and platform-independent.
        - Represents data as **key-value pairs** and supports complex structures like objects and arrays.
    - **XML**:
        - Hierarchical tree structure.
        - More verbose and complex compared to JSON.
- **JSON Structure**:
    
    - **Key-Value Pairs**: Basic unit in JSON, with keys as strings and values being various data types (strings, numbers, objects, arrays, booleans, or null).
    - **Objects**: Defined by curly braces `{}`, containing key-value pairs.
    - **Arrays**: Defined by square brackets `[]`, containing ordered lists of values.
    - **Nested Structures**: JSON allows objects and arrays to be nested within each other, enabling complex data representations.
- **JSON Data Types**:
    
    - **Strings**: Text enclosed in double quotes `" "`.
    - **Numbers**: Integer or floating-point values.
    - **Objects**: Another set of key-value pairs enclosed in `{}`.
    - **Arrays**: Ordered collection of values in `[]`.
    - **Booleans**: `true` or `false`.
    - **Null**: Represents an empty or undefined value.
- **Processing JSON**:
    
    - Before processing JSON, you need to understand the **object hierarchy** and the **attributes** (keys) of each object.
    - For example, a **PersonInfo** object might contain a **first_name**, **last_name**, **address** (which is another object), and **phone** (which could be an array).
    - Data structures in programming languages, such as **NSDictionary** for key-value pairs and **NSArray** for arrays, are often used to represent JSON data in iOS.
- **Key Processing Steps**:
    
    1. Identify the objects and their attributes.
    2. Recognize the data type for each attribute.
    3. Define the data structure (e.g., PersonInfo with nested Address).
    4. Process and extract relevant information from the JSON structure.
