### Software Design in Robotics and Autonomous Systems

- **Goal**: Organize computational requirements into a structured software architecture.
- **Key Components**:
    - **Data Acquisition Layer**: Captures data from sensors.
    - **Processing Layer**: Interprets sensor data and makes decisions.
    - **Actuator Control Layer**: Manages actuators for executing actions.

### Layered Architecture Overview

1. **Data Acquisition (Sensor Layer)**:
    
    - Abstraction for sensors like radars, LIDARs, cameras, and GPS.
    - Captures external (environmental) and internal (vehicle state) data.
2. **Processing Layer**:
    
    - **Sensor Fusion**: Merges sensor data, removes noise, increases accuracy.
    - **World Model**: Generates an external environment model, often a “bird’s eye view,” including vehicle position and surroundings.
    - **Behavior Generator**: Uses Finite State Models to decide actions (e.g., braking).
    - **Planning**: Maps out each maneuver for safety, collision avoidance, etc.
    - **Vehicle Control**: Executes planned tasks, e.g., adjusting brakes or steering.
3. **Actuator Control Layer**:
    
    - Translates control signals to actuator actions (e.g., steering, braking).
    - Receives signals from the processing layer to carry out physical actions in the vehicle.

### Feedback Loops and Dynamic Adjustments

- **Feedback Mechanisms**:
    - **Sensor Adjustment**: Feedback to sensors for calibration (e.g., adjusting speed or accuracy).
    - **Actuator Adjustment**: Modifies control inputs based on hardware conditions (e.g., more braking force for worn brakes).
    - **Purpose**: Self-learning and self-correction to adapt operations based on system state.

### Self-Driving Car Software Architecture Example

- **Proposed by IEEE International Conference on Software Architecture**:
    
    - **Key Layers**:
        - **Sensor Abstraction**: Interface for various sensors.
        - **Actuator Abstraction**: Interface for actuators (steering, brakes).
        - **Data Processing Components**:
            - **Sensor Fusion**: Integrates sensor data.
            - **World Model**: Maps vehicle position in the environment.
            - **Behavior Generator**: Determines appropriate vehicle behavior.
            - **Planning**: Outlines vehicle actions based on selected behavior.
            - **Vehicle Control**: Directly controls actuators.
- **Support Layers**:
    
    - **Data Management Layer**: Stores historical data for future decision-making.
    - **System Safety Management Layer**: Ensures safety by monitoring internal and external conditions.

### Key Data Types and Roles

- **Internal Data**: Vehicle state (e.g., speed, hardware status).
- **External Data**: Environmental information for navigation and safety.
- **Importance of Data Management**:
    - **Purpose**: Facilitates persistent storage for state-based actions and self-learning.
    - **Examples**: Maintains knowledge of previous states to make contextually informed decisions.

### Safety Management in Autonomous Systems

- **Critical Role**: Compliance with safety regulations and internal safety checks.
- **Safety Management Layer**: Manages protocols, handles safety features, and provides continuous monitoring.

### Summary and Key Takeaways

- **Autonomous System Software Challenges**: Multi-layered architecture with various abstractions and components for data processing, safety, and control.
- **Framework Flexibility**: Adapts to vehicle type or autonomous system requirements.
- **Published Architecture**: IEEE’s framework serves as a generic model with customizable layers for specific needs.
