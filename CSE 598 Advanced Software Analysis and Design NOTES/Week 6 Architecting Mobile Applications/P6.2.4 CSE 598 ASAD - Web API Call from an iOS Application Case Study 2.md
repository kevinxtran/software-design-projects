#Week6
1. **API Call and JSON Handling**:
    
    - The case study involves making an API call to `newsapi.org`, which provides news data in JSON format.
    - The API returns various fields for each news item, such as the title, author, description, publication date, and URL.
    - **Important Structure**: The JSON response contains an `articles` array. Each article is an object with fields like `author`, `title`, `publishedAt`, `url`, and `urlToImage`.
2. **Table View Application**:
    
    - The goal is to create a Table View to display a list of news articles. The Table View will display the title and author for each article.
    - When a user clicks on a row, a segue will load the detailed webpage for that news item.
3. **Key iOS Concepts**:
    
    - **Segue**: A segue is used to transition from one view controller to another. In this case, selecting a news item in the Table View triggers a segue to load a web page with the news details.
    - **Prototype Cell**: A prototype cell in the Table View acts as a template for rows. It is configured with a title and subtitle (news title and author).
4. **UI Design (Storyboard)**:
    
    - The UI is designed using **Storyboard**. It consists of:
        - A **Table View** in the main view.
        - A **prototype cell** inside the Table View to display the news data.
        - A **segue** linking the Table View to a second view controller that displays the web page for the selected news item.
    - The main and second view controllers are linked through a **segue** called `detailNewsSegue`.
5. **Swift Data Structures**:
    
    - To decode the JSON response, two main data structures are used:
        - **News**: A decodable object containing an array of articles (`articles`).
        - **Article**: Another decodable object representing individual news items, with fields for the title, author, description, and URL.
    - Both objects conform to the `Decodable` protocol to allow for easy conversion of the JSON into Swift objects.
6. **Asynchronous API Call**:
    
    - The API call is made asynchronously to avoid blocking the main UI thread. This allows the user to interact with the app while waiting for the API response.
    - **DispatchQueue.main.async**: This ensures that after decoding the JSON and getting the news articles, the table view is reloaded on the main thread.
7. **Decoding JSON into Swift Objects**:
    
    - After making the API call and receiving the JSON data, the data is decoded into Swift objects using `JSONDecoder`.
    - The decoded data is stored in an array (`newsItems`), which holds the list of articles.
    - The main UI is updated by reloading the Table View to display the articles.
8. **UI Updates**:
    
    - Once the articles are decoded, the Table View is reloaded using `reloadData()`, which refreshes the UI with the new data.
9. **Handling API Key**:
    
    - To use the news API, you need to register and obtain an API key. This key is included in the request URL to authenticate the API calls.
10. **Asynchronous Method**:
    
    - The asynchronous call ensures that the UI remains responsive while waiting for the data from the API.
    - When the data is ready, the main thread is updated to reload the Table View with the new data.
11. **Important iOS Patterns**:
    
    - **ViewControllers**: Each view is controlled by a `ViewController` class (e.g., `ViewController.swift` for the main view and `NewsViewController.swift` for the detail view).
    - **Updating the UI**: Always update the UI on the main thread, especially when dealing with asynchronous tasks like network requests.

### Study Recommendations:

- **Understand JSON Decoding**: Familiarize yourself with how to decode JSON responses into Swift data structures using `Decodable` and `JSONDecoder`.
- **Segues and Table Views**: Focus on how segues work to navigate between views and how Table Views populate with dynamic data (in this case, news items).
- **Async API Calls**: Practice making asynchronous API calls and updating the UI based on the response.
- **UI Design**: Learn how to design and configure Table Views and segue connections in Storyboard.
