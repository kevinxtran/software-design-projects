### 1. **Unified Modeling Language (UML) Overview**

- **Purpose**: UML is a standard modeling tool used in object-oriented systems to represent objects and relationships.
- **Focus**: Modeling of individual objects, their attributes and behaviors, and how they interact to represent the system's static structure.
- **Diagrams Covered**: Class diagrams specifically, which are central for transitioning from analysis to coding.

### 2. **Class Diagrams**

- **Definition**: Visual representation of objects (classes) and the relationships among them, showcasing the static structure of a system.
- **Components**:
    - **Classes**: Represent individual objects.
    - **Relationships**: Links between classes showing how they interact or depend on each other.
- **Importance**: First step in technical documentation with a coding perspective.

### 3. **UML Notation in Class Diagrams**

- **Class Representation**: Three main components:
    1. **Class Name**: Name of the object (e.g., Car).
    2. **Attributes**: Characteristics of the object, marked as private (denoted with a “-” sign), with data types specified (e.g., `-make: String`).
    3. **Methods/Behaviors**: Actions the object can perform, typically public (denoted with a “+” sign), such as `+accelerate()` or `+stop()`.

### 4. **Object Attributes and Behaviors**

- **Attributes**: Represent properties of the object (e.g., `make`, `model`, `color`).
- **Behaviors**: Define what an object can do, public methods that others can interact with (e.g., `accelerate()`, `brake()`).

### 5. **Types of Relationships in UML**

- **Main Categories**:
    1. **Structural Relationships**: Closer ties, often involving data or method access.
    2. **Non-Structural Relationships**: Weaker connections, primarily one object calling another to perform a task.

### 6. **Detailed Relationship Types**

- **Dependency (Non-Structural)**:
    - **Definition**: One object calls another's method, relying on it to perform specific actions.
    - **Example**: An `OnlineStore` depends on a `PaymentModule` for processing payments.
- **Inheritance / Generalization (Structural)**:
    - **Definition**: "Is-a" relationship; one class derives from another, sharing general traits but with specific differences.
    - **Example**: `SpecialOrder` and `RegularOrder` both inherit from `Order`.
    - **Notation**: Arrow from child (derived class) to parent (base class).
- **Association (Structural)**:
    - **Definition**: One object needs access to another to function.
    - **Example**: `OnlineStore` accesses `Order` to process customer requests.
    - **Multiplicity**: Indicates quantity of the association, e.g., one store can process multiple orders.
- **Aggregation (Structural)**:
    - **Definition**: "Has-a" relationship where one object is a part of another but can exist independently.
    - **Example**: `OnlineStore` has multiple `SalesRepresentatives`.
    - **Notation**: Empty diamond on the side of the whole (e.g., `OnlineStore`).
- **Composition (Strong Structural)**:
    - **Definition**: Stronger "contains-a" relationship where part cannot exist independently of the whole.
    - **Example**: `Order` consists of multiple `OrderItems`, which cannot exist separately.
    - **Notation**: Filled diamond at the whole side (e.g., `Order`).

### 7. **UML Notation Recap for Relationships**

- **Dependency**: Weak link; one object uses another to complete a task.
- **Inheritance/Generalization**: Is-a relationship with shared characteristics.
- **Association**: Direct access relationship; used for processing or accessing data.
- **Aggregation**: Whole-part relationship; parts can exist without the whole.
- **Composition**: Strong whole-part relationship; part’s lifecycle is tied to the whole.

### 8. **Building the Class Diagram**

- **Combining Components**: Class names, attributes, methods, and relationships combine to form the complete class diagram.
- **Purpose**: Establishes a visual representation of how all objects and relationships interact to form the system’s static structure, essential for coding.
