#Week5 
- **Three-Party Model of Web Services**:
    
    - Client Applications: These can be mobile, desktop, or web applications.
    - Communication Network: Serves as the medium for communication between the client and web services.
    - Web Services: Can be either SOAP or REST-based, which interact with client applications.
- **Language and Platform Independence**:
    
    - Client applications can use either SOAP or REST Web Services, regardless of the client or server-side technology (e.g., .NET, Java, Python, JavaScript).
    - This flexibility is due to the loosely coupled nature of web services. It means that the client’s technology stack doesn’t need to match the web service’s implementation.
- **Loosely Coupled Nature**:
    
    - Client and web service developers can work independently and at different times.
    - Applications can be deployed across diverse hardware and software platforms, as long as they have internet access and can communicate via HTTP or SOAP.
- **Service Brokers and Registries**:
    
    - Web service developers announce the availability of their services via service registries (e.g., Programmable Web).
    - Application developers can search these registries to find relevant web services, including API URLs and descriptions of functionality.
- **Example of a Web Service Registry**:
    
    - Programmable Web (programmableweb.com) lists over 16,000 APIs for various services like weather, directions, and news.
    - Other providers (e.g., Google, Microsoft) offer APIs that can be used by application developers.
- **Developing a Web Client Application**:
    
    - **Technology Stack**: In the example, .NET Framework and C# are used, but similar technologies can be applied with Java, Python, etc.
    - **Creating the Web Application**:
        1. Start a new project in Visual Studio and select the ASP.NET Website application template.
        2. Add a web form and design the UI using the designer view, adding controls like labels, textboxes, and buttons.
        3. Attach actions to buttons, for example, calling a web service upon clicking a button.
- **SOAP Web Service Integration**:
    
    1. **WSDL URL**: The URL of the SOAP service’s WSDL (Web Services Description Language) document is needed.
    2. **Proxy Object**: A proxy object for the SOAP service is created using the WSDL, which allows interaction with the SOAP web service.
    3. **Invoking the SOAP Web Service**:
        - Extract input from the textbox.
        - Call the SOAP service method (e.g., `computeStat`) and pass the input string.
        - Process the result, which is returned as an object (deserializable) and extract relevant information (e.g., digits).
    4. **Display the Result**: The processed result is shown on the UI.
- **REST Web Service Integration**:
    
    1. **Forming the URL**: Construct the URL for the REST service using the base URL, the API endpoint (e.g., `/computeStatJSON`), and parameters (e.g., the input string).
    2. **HTTP Request**: Create an HTTP web request object and use it to invoke the URL.
    3. **Processing the Response**:
        - The response is returned as a stream (typically JSON or XML).
        - Convert the stream to a string (e.g., JSON text).
        - Deserialize the JSON string into an object to extract specific data (e.g., number of digits).
    4. **Display the Result**: The processed result is displayed to the user.
- **Handling Complex Data Types**:
    
    - **Serializable Objects**: For both SOAP and REST, it’s important to define the expected result object structure on the client side (e.g., digits, uppercase letters).
    - This allows proper deserialization of the service response and enables access to specific fields (e.g., `.digits`).
- **Main Steps for Creating a Web Client for Web Services**:
    
    1. Design the UI and add appropriate controls (labels, textboxes, buttons).
    2. Implement event handlers (e.g., button click) to invoke the web service.
    3. For SOAP services, use WSDL to generate the proxy object and invoke the method.
    4. For REST services, form the URL and use HTTP requests to access the service, then process the response (JSON/XML).
    5. Deserialize the response into an object, extract needed values, and display them on the UI.
