#### Note: 
- These concepts are crucial for understanding how iOS applications interact with Web APIs, handle JSON data, and update UI elements asynchronously.
#### 1. **Web API Basics**:

- The case study discusses how an iOS application makes a Web API call and processes the returned JSON data.
- The Web API, `computeStatJson`, accepts a string and returns a JSON object containing:
    - Count of uppercase letters
    - Count of lowercase letters
    - Count of digits in the string.

#### 2. **UI Design and Elements**:

- The user interface (UI) consists of:
    - A text field (`UITextField`) where the user inputs a string.
    - A button (`UIButton`) that triggers the API call when clicked.
    - A text view (`UITextView`) to display the results.
- **IBOutlet**: UI elements are connected to code through `IBOutlet`, allowing data to be read or updated programmatically.
- **IBAction**: The button has an `IBAction` to trigger the process, sending data from the text field to the Web API and updating the UI with the results.

#### 3. **Processing JSON Data**:

- **JSON Structure**: The Web API returns data in JSON format with three key-value pairs:
    - `digits`: Integer representing the count of digits in the string.
    - `lowercase`: Integer representing the count of lowercase letters.
    - `uppercase`: Integer representing the count of uppercase letters.
- **Decodable Objects**: The JSON structure is mapped to a `Decodable` Swift object called `mathResults`, which holds integer values for digits, lowercase letters, and uppercase letters.
- The `Decodable` object ensures that the structure of the JSON response matches the Swift object for easy processing and access.

#### 4. **Making the API Call**:

- The user input is used to create a dynamic URL for the API call (e.g., appending the input string as a parameter).
- The **URLSession** class is used to make the network request:
    - `urlSession.dataTask` is invoked to get the data from the API asynchronously.
    - **Completion Handler**: The network request uses a completion handler to handle the response asynchronously. This means other tasks can proceed while waiting for the API response.

#### 5. **Decoding JSON Data**:

- **JSONDecoder**: After receiving the data, it's decoded using `JSONDecoder` to convert the raw JSON data into a `mathResults` object.
- The decoding process ensures the data types match between the JSON keys and the Swift properties.

#### 6. **Updating the UI**:

- After decoding the JSON into the `mathResults` object, the UI is updated by:
    - Accessing the decoded data using dot notation (e.g., `jsonResult.digits`).
    - Constructing a result string (e.g., concatenating the counts of uppercase, lowercase, and digit characters).
- **Main Queue**: UI updates should be done on the main thread to ensure the UI reflects changes correctly. The `DispatchQueue.main.async` method is used to perform the UI update asynchronously.

#### 7. **Asynchronous Processing**:

- The API call and JSON decoding are asynchronous, allowing the app to remain responsive while waiting for the API results.
- The results are processed once the data is retrieved and decoded, and the UI is updated accordingly on the main thread.

#### 8. **Key Constructs and Process**:

- **API Call**: Identifying the correct API and understanding its parameters (e.g., string type input).
- **JSON Structure**: Analyzing the JSON response to understand its structure (key-value pairs).
- **Decodable Object**: Creating a `Decodable` object in Swift that matches the JSON structure.
- **API Call & Processing**: Making the API call using `URLSession`, decoding the data with `JSONDecoder`, and updating the UI using the main thread.

#### 9. **Additional Considerations**:

- Ensure the keys in the JSON structure match exactly with the variable names in the `Decodable` struct.
- The entire process involves:
    - Collecting user input.
    - Making a network request.
    - Decoding JSON data.
    - Updating the UI with the processed data.
