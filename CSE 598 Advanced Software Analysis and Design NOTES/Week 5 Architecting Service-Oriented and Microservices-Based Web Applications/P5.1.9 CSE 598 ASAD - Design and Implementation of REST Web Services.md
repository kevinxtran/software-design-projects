**Overview of Developing a RESTful Web Service:**

1. **Basic Principles & Steps for Developing a RESTful Web Service:**
    
    - **Requirement Analysis**: Understand the purpose of the web service and what functionalities it will support.
    - **Define the Service**: Specify the interface and methods for the service.
    - **Create Web Service Project**: Create a project to implement the service.
    - **Define & Implement Methods**: Define service methods and implement them in the appropriate files.
    - **Deploy**: Deploy the service to a server for use.
2. **Steps in .NET Framework for REST Web Services:**
    
    - Use **WCF Service Application** to create the project.
    - Define methods in **IServices.cs** (interface file) and implement them in **Services.cs** (implementation file).
    - The service can be tested using **IIS Server** in Visual Studio.
3. **REST vs SOAP Web Services:**
    
    - **SOAP**: Uses an envelope to define the structure and relies on SOAP protocol.
    - **REST**: Uses HTTP directly for communication, and methods are invoked via HTTP requests.

**Math Web Service Example:**

1. **Web Service Methods:**
    
    - Methods include **Absolute Value**, **PI Value**, **Compute Stat JSON**, **Compute Stat XML**, and **Sum**.
    - REST services return results as JSON or XML, whereas SOAP would return them in objects.
    - For example, **compute stat JSON** returns JSON, and **compute stat XML** returns XML.
2. **Method Invocations:**
    
    - Invoke methods via URLs using HTTP protocols (GET method) to access the service:
        - **Example URL**: `/computeStatJson?param=helloworld`
        - Depending on the method, the response could be in JSON or XML format.

**Steps for Creating a REST Web Service Project in .NET Framework:**

1. **Create a New Project in Visual Studio:**
    
    - Select **Visual C#** and **WCF Service Application**.
    - Use the default template to generate **IServices** and **Services.cs**.
    - Rename files if necessary (e.g., **IMathService.cs**, **MathService.cs**).
2. **Interface Definition:**
    
    - Use **Service Contract** to define the service interface.
    - Define methods as **Operation Contracts**.
    - Methods can be accessed using HTTP methods like GET, POST, PUT, DELETE.
    - **WebGet** is used to fetch data (GET method).
    - **WebInvoke** is used for operations that modify data (POST, PUT, DELETE).

**Details of Defining HTTP Methods:**

1. **GET vs POST vs PUT vs DELETE (CURD Operations):**
    
    - **GET**: Retrieves data from the server (e.g., **abs value** or **pi value**).
    - **POST**: Used to send data to the server for creation (e.g., insert data).
    - **PUT**: Used to update existing data.
    - **DELETE**: Used to delete data.
2. **WebGet and WebInvoke Usage:**
    
    - **WebGet**: Used with HTTP GET method to retrieve data.
        - Can specify **ResponseFormat** (e.g., JSON or XML).
    - **WebInvoke**: Used for methods requiring actions like inserting or updating.
        - Allows specifying the HTTP method (GET, POST, PUT, DELETE).
        - **WebMessageFormat** defines the response format (e.g., JSON or XML).
3. **Data Contracts for Complex Types:**
    
    - Data Contracts define complex return types, such as the output of the **Compute Stat JSON** method, which includes counts for digits, uppercase, and lowercase letters.
    - **DataContract** is a class where data members are defined (e.g., `digits`, `uppercase letters`, `lowercase letters`).

**Return Type and Format:**

1. **Response Format (JSON/XML):**
    - You can set the **ResponseFormat** to either **WebMessageFormat.JSON** or **WebMessageFormat.XML**.
    - Based on the format set, the service will return either a JSON or XML document.

**Conclusion:**

- To implement RESTful web services in **.NET Framework** using **C#**:
    - Use **WebGet** for GET requests to retrieve data.
    - Use **WebInvoke** for other HTTP methods like POST, PUT, DELETE.
    - Define the **ResponseFormat** to determine if the response is in JSON or XML format.
    - For complex return types, use **DataContracts** to define the data structure.
