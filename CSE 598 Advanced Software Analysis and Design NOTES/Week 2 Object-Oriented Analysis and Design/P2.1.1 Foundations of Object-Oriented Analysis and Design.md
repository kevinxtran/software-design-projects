**Foundations of Object-Oriented Analysis and Design**

1. **Object Orientation Basics:**
    
    - Object-oriented programming (OOP) focuses on the use of _objects_ to represent entities in a system.
    - Objects consist of:
        - _Data_ (fields, attributes, or states)
        - _Methods_ (behaviors or operations)
2. **Real-World Analogy:**
    
    - A playground analogy is used to explain OOP:
        - The playground is a system with objects like swings, slides, and roundabouts.
        - Each object has its own properties (e.g., swing height, weight capacity) and functionality (e.g., swinging motion).
    - Similar to a playground, OOP systems consist of objects with defined functionalities and attributes that interact with one another to perform tasks.
3. **Object Characteristics:**
    
    - Objects contain _related data_ and _methods_ in a single unit.
    - At run-time, objects communicate with one another to perform operations, similar to how different playground structures serve multiple activities.
4. **Designing Objects:**
    
    - **[[Abstraction]]**:
        - Objects are modular and abstract collections of related data and procedures.
        - Modular design ensures that objects can be independently modified without affecting the system (e.g., replacing a swing without modifying the entire playground).
    - **Encapsulation**:
        - Data and methods are combined within an object, keeping the internal data hidden from external programs.
        - External access is provided through methods (interfaces) only.
    - **Information Hiding**:
        - Limits the exposure of internal object details to minimize unintended side effects.
        - Changes inside an object do not affect how external programs interact with it due to encapsulation and controlled interfaces.
5. **Three Main Concepts of Object-Oriented Design:**
    
    - **Abstraction**: Separating the implementation details of objects and focusing on their essential attributes and behaviors.
    - **Encapsulation/Modularity**: Combining data and methods within an object, ensuring that internal details are hidden from the outside.
    - **Architecture**: The overall organization of objects in a system and how they communicate for optimal performance.
6. **Data and Procedural Abstractions:**
    
    - **Data Abstraction**: Describes what an object is.
        - Example: A car object may include attributes like make, model, color, engine power, etc.
    - **Procedural Abstraction**: Describes what an object can do.
        - Example: A car object can start, stop, turn, accelerate, and reverse.
    - Together, data and methods are combined in an object, forming a "black box" where data is manipulated via methods, and external access is controlled through the interfaces (methods).
7. **Information Hiding Benefits:**

    - Reduces the risk of side effects from global data.
    - Maintains the stability of external interfaces even when internal changes occur.
    - Emphasizes communication through well-defined interfaces.
8. **Example Object: Light Bulb**

    - _Attributes_: Brightness, color
    - _Methods_: Switch on, switch off, dim
    - Objects like a light bulb abstract both their data and behaviors, providing a way to encapsulate the core features of the object.
