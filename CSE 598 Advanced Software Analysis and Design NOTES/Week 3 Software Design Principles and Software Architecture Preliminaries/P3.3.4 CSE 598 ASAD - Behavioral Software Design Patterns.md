#Week3
#### Overview:

- **Behavioral design patterns** focus on the communication patterns between objects, helping to define how objects interact in a system.
- Commonly used behavioral patterns: **Observer**, **Strategy**, **Iterator**, **State**, **Visitor**, **Mediator**, **Chain of Responsibility**.

#### 1. **Chain of Responsibility**

- **Purpose**: Pass requests through a chain of objects to fulfill the request.
- **Client**: The client makes the request to the first object in the chain, which then passes it down the chain until the request is fulfilled by the appropriate object.
- **Example**: Java Logger System:
    - The logger sends log requests to various handlers, which then filter and modify the request before posting it to external systems.
- **Benefit**: This pattern decouples the sender and receiver of the request, allowing for flexible handling of requests.

#### 2. **Iterator**

- **Purpose**: Allows sequential access to elements in a collection without exposing the collection's internal structure.
- **How it Works**: An **iterator** provides methods (e.g., `next()`, `previous()`) to traverse through a collection.
- **Example**: Java LinkedList
    - An iterator for a `LinkedList` allows navigation through the list, adding or removing elements.
- **Benefit**: Simplifies accessing elements in a collection, especially if the underlying structure is complex.

#### 3. **Observer**

- **Purpose**: Defines a notification mechanism where changes to one object notify other registered objects.
- **How it Works**: Objects (observers) subscribe to notifications from another object (subject). When a change occurs in the subject, observers are notified and can react accordingly.
- **Example**: Java ActionListener in GUI (e.g., button click events).
    - An `ActionListener` is attached to a `JButton`. When the button is clicked, the listener is notified and performs a task.
- **Benefit**: Useful in GUI systems and any application that requires event-driven behavior or backend notification systems.

#### 4. **State**

- **Purpose**: Alters an object's behavior when its state changes.
- **How it Works**: Different states of an object are represented by separate classes or methods. Behavior changes based on the current state.
- **Example**: Elevator system
    - An elevator behaves differently when idle, moving up, or moving down. Each state triggers different responses to requests.
- **Benefit**: Helps manage objects with multiple states, improving modularity and cohesion.

#### 5. **Strategy**

- **Purpose**: Allows the selection of an algorithm at runtime.
- **How it Works**: A collection of algorithms is encapsulated as objects. The system can choose the appropriate algorithm at runtime based on the context.
- **Example**: Polymorphic strategies (i.e., algorithm selection at runtime using polymorphism).
- **Benefit**: Facilitates flexibility and scalability by allowing runtime decision-making about which behavior or algorithm to apply.

#### 6. **Mediator**

- **Purpose**: Provides a communication channel to mediate interactions between objects, reducing direct dependencies.
- **How it Works**: Objects send messages to the mediator (a communication bus) rather than directly to one another. The mediator handles communication between objects, preventing tight coupling.
- **Example**: A system where multiple objects (A, B, C) need to communicate, but instead of each object talking directly to the others, they use a mediator to communicate, isolating the interactions.
- **Benefit**: Reduces complexity and tight coupling between objects, improving maintainability and flexibility.

#### General Benefits of Behavioral Design Patterns:

- **Communication**: Facilitate efficient and clear interaction between objects.
- **Modularity**: Improve modularity by decoupling objects and their interactions.
- **Cohesion**: Enhance cohesion by ensuring that each object’s behavior is encapsulated based on its state or strategy.
- **Low Coupling**: Reduce coupling between objects by managing interactions through patterns like the mediator, strategy, and observer.
