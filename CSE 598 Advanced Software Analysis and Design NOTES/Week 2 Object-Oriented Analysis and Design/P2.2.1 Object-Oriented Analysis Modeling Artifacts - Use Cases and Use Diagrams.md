### Key Concepts in Object-Oriented Analysis Modeling

1. **Objective**: Focus on object-oriented analysis modeling artifacts, specifically **use cases** and **use case diagrams**.

### 1. Object-Oriented Analysis Main Approach

- **Requirement Analysis**: Analyze software requirements from different perspectives to create software engineering artifacts.
- **Scenario-Based Analysis**: Begin with user interactions to identify how these interactions function in operational scenarios.
- **Artifact Creation**: Convert user requirements into artifacts like class diagrams and identify objects.

### 2. Use Case Identification Process

- **Actors**: Entities (users or external systems) interacting with the system.
- **Use Cases**: Main functionalities of the system.
- **Use Case Scenarios**: Detailed steps of how each use case functions.
- **Use Case Diagram**: Illustrates the relationship between actors and use cases, showing a high-level view of system functionality.

### 3. Iterative Process of Refining Use Cases

- **Refinement**: Use cases and actors are refined iteratively to improve the use case diagram’s accuracy.

### 4. Types of Relationships in Use Case Diagrams

- **Extends**: Optional functionalities (e.g., a welcome message after user validation).
- **Includes**: Required functionalities (e.g., validating the customer before deposit in an ATM).
- **Generalization**: Common functionalities shared across use cases (e.g., transactions for ATM operations like deposit and withdraw).

### 5. Example - ATM Use Case Diagram

- **Actors**: Bank customer.
- **Use Cases**: Withdrawal, deposit, etc.
- **Relationships**:
    - _Include_: Validate PIN included in transactions.
    - _Generalization_: Transactions are a general class for deposit and withdraw operations.

### 6. Writing Detailed Use Case Scenarios

- **Template Components**:
    
    - **Name**: Unique name of the use case.
    - **Description**: Main purpose of the functionality.
    - **Actors**: Entities involved in the use case.
    - **Basic Flow of Actions**: Sequence of actions that describe how the use case operates.
- **Example**: ATM Withdraw Money use case involves:
    
    - Inserting the card, PIN validation, selecting the amount, verifying balance, and dispensing cash if criteria are met.

### Summary of Workflow

- Identify actors and use cases → Create use case diagram → Refine and establish relationships → Identify objects and relationships for CRC (Class Responsibility Collaborator) diagrams.
