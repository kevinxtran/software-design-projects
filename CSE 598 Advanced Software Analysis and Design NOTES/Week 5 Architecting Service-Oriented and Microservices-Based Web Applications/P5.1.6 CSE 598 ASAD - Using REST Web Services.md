#Week5 
### Overview

- **Goal**: Use a third-party RESTful web service within a .NET client application using Visual Studio and C#.
- **RESTful Web Services**: Accessed through HTTP protocol, these services are widely compatible and typically return results in formats like JSON, XML, or basic data types.

### Steps to Develop a Client Application for REST API

1. **Create a Client Project**:
    
    - Use Visual Studio to create a **Console Application** in C#.
    - This app will serve as the client to make API requests and process responses.
2. **Set Up API Request**:
    
    - Use either `HttpWebRequest` or `HttpClient` for the REST API call.
    - For this example, `HttpWebRequest` is used.
    - The API's **URL** is embedded in the code, specifying:
        - **Service Layer URL**: The server location.
        - **API Endpoint**: Specifies the method being called (e.g., `abs value` for an absolute value calculation).
        - **Parameters**: Any inputs needed by the API (appended to the URL).
3. **Invoke the API**:
    
    - Form a **web request** with `WebRequest.Create(URL)`.
    - Call the `GetResponse()` method to execute the HTTP request.
    - **Response Handling**: Convert the response (in byte stream) to a readable format using `StreamReader`.
4. **Process and Display Response**:
    
    - **Basic Data**: Directly display if the response is a simple type (e.g., integer).
    - **JSON or XML**:
        - **JSON**: Parse as an object and access values by key.
        - **XML**: Use XML parsing (e.g., an XML reader) to retrieve data from nested nodes.

### Example of RESTful Web Service (Math API)

1. **API Methods**:
    
    - `pi value`: Returns the constant π, takes no parameters.
    - `abs value`: Returns the absolute value of a given integer.
    - `ComputeStat JSON` / `ComputeStat XML`: Returns text statistics (e.g., counts of digits, uppercase/lowercase) in JSON or XML format.
    - `int sum`: Takes two integers, returns their sum.
2. **Example URL Formation**:
    
    - **Structure**: `Service URL` + `API Name` + `Parameters`
    - **Example**: `/MathService/abs?value=-10` invokes the absolute value API with `-10` as the input.

### Detailed Explanation of JSON and XML Handling

1. **JSON Format**:
    
    - JSON is often simpler to parse, consisting of key-value pairs.
    - Example:
        
        json
        
        Copy code
        
        `{   "digits": 3,   "lowercase": 8,   "uppercase": 2 }`
        
    - Process with JSON parsers to convert into an object and access individual values.
2. **XML Format**:
    
    - XML structure has hierarchical tags and requires an XML reader for parsing.
    - Example:
        
        xml
        
        Copy code
        
        `<result>   <digits>3</digits>   <lowercase>8</lowercase>   <uppercase>2</uppercase> </result>`
        
    - Parsed into a tree structure, with each tag (node) accessible for data extraction.
3. **Complex JSON**:
    
    - APIs like weather services may return JSON with nested arrays and objects.
    - **Curly brackets `{}`** denote objects; **square brackets `[]`** denote arrays.
    - Example:
        - **Weather**: Array of multiple weather data objects.
        - Supports complex data types, including integers, strings, Booleans, and arrays of objects.

### Advantages and Limitations of RESTful Web Services

1. **Advantages**:
    
    - **Widely Accessible**: HTTP protocol allows access across nearly all web-compatible devices.
    - **Simplicity**: HTTP makes API calls relatively straightforward and accessible.
2. **Limitations**:
    
    - **Protocol Dependency**: RESTful services are limited to HTTP; for other protocols, additional configurations may be needed.

### Practical Summary of Client-Server Workflow

1. **Client Side**:
    
    - Embed the URL in the client program.
    - Make the API call, handle the response, and display or process data as needed.
2. **Server Side**:
    
    - Receives the API call, processes it, and returns results in JSON, XML, or a basic data type.
    - The client can then utilize these results for computations or display them directly to users.
